Chapter 10: Virtual Memory

Virtual memory lets a process behave as if it has more memory than the installed RAM. The big ideas here are demand paging, page faults, replacement algorithms, and thrashing.

1. What Virtual Memory Means

  • Virtual memory separates logical memory from physical memory.
  • Only part of a program must be in RAM for execution.
  • This allows the logical address space to be larger than physical memory.
  • It also allows more efficient process creation and lets more programs run at the same time.

2. Demand Paging and Page Faults

  • Demand paging means a page is loaded only when it is actually needed.
  • When the CPU generates a virtual address, the MMU checks the page table.
  • If the page is marked invalid because it is not in RAM, a page fault occurs.
  • The OS then loads the needed page from disk into RAM and restarts the interrupted instruction.

3. Dirty Bit and Page Replacement Setup

  • The dirty bit tells whether a page was modified after being loaded.
  • If the chosen victim page is dirty, it must be written back to disk before removal.
  • If it is clean, it can be discarded without extra write cost.
  • This improves performance because the system avoids unnecessary disk writes.

4. Basic Page Replacement

  • If a free frame exists, the needed page is simply loaded there.
  • If RAM is full, the OS must choose a victim page to replace.
  • After replacement, the page table and frame information are updated.
  • The process then continues from the instruction that caused the fault.

5. Page Replacement Algorithms

AlgorithmMain RuleImportant Point
FIFOReplace the oldest pageSimple, but can show Belady's anomaly
OPTReplace the page used farthest in the futureBest possible, but theoretical
LRUReplace the least recently used pageUses past behavior and is often practical
LFUReplace the least frequently used pageBased on access count
MFUReplace the most frequently used pageLess common

6. What the Algorithms Really Mean

  • FIFO is easy to implement but not always intelligent.
  • OPT is the benchmark because no real algorithm can do better on the same reference string.
  • LRU assumes pages used recently will likely be used again soon, so it often performs well.
  • Belady's anomaly means FIFO can sometimes produce more page faults when the number of frames increases.

7. Thrashing

  • Thrashing means the process spends more time paging than actually executing instructions.
  • CPU utilization becomes low because the system is busy moving pages between RAM and disk.
  • The OS may make the problem worse if it incorrectly increases the degree of multiprogramming.
  • A common fix is to ensure each process has enough frames, for example by watching page-fault frequency.
Best quick summary: virtual memory lets the system run larger or more programs, but too many page faults can collapse performance into thrashing.

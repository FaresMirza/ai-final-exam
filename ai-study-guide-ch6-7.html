Chapter 6

Unsupervised Learning — Dimensionality Reduction

What "Dimensions" Mean Here

A dimension is just a feature/column in your data. A dataset with 100 columns lives in "100 dimensions." Dimensionality reduction means squeezing many columns down into a few while keeping the important information.

Easy example

Describing a person with 50 facts (height, weight, age, hobbies…). Most of who they are can be captured in just a few key facts. You keep the few that matter and drop the rest.

Supervised vs Unsupervised (quick reminder)

Supervised

  • Has labels (answers)
  • Learns input → known output
  • Ch 2 & 3 were this

Unsupervised

  • No labels
  • Finds hidden structure on its own
  • This chapter is this
Remember

Dimensionality reduction is unsupervised — there's no answer key. It just reorganizes the data into a smaller, cleaner form.

The Curse of Dimensionality

The big problem this chapter solves: when data has too many features, models get worse, slower, and harder to trust.

  • Data sparsity — points spread so far apart that "neighbors" become empty. Patterns disappear.
  • Distance concentration — everything ends up roughly the same distance apart, so "nearest" stops meaning anything.
  • Computational explosion — more features = way more memory and time.
Easy example

Finding a friend in a small room is easy. In a giant empty stadium, everyone is "far away" and equally hard to reach. Extra space (dimensions) makes everything feel spread out and similar.

Remember

More features ≠ better. Past a point, extra features add noise, cause overfitting, and make data impossible to visualize (we can only see 2D/3D).

The Two Goals

  • Fewer features — cut the number of columns down.
  • Preserved structure — but keep the meaningful relationships (what's similar to what).

There are three families of methods: Linear (PCA), Nonlinear (t-SNE, UMAP), and Neural (Autoencoders).

PCA (Principal Component Analysis)

The most popular method. It finds the directions where the data varies the most and keeps those, dropping the boring low-variance directions. It's linear (works with straight-line relationships) and deterministic (same input → same output every time).

  • Maximize variance — keep the directions that carry the most information.
  • Remove redundancy — merge features that secretly say the same thing.
Easy example

Photographing a chair. You instinctively pick the angle that shows the most detail, not a flat side that hides its shape. PCA picks the "angles" (directions) that reveal the most about your data.

Remember — the "elbow" rule

Plot how much information each component adds. Pick the number at the elbow, where adding more barely helps. Often a few components capture 90%+ of the info.

PCA — Strengths & Limits

Strengths

  • Fast & scalable
  • Deterministic (repeatable)
  • Interpretable

Limits

  • Linear only
  • Misses curved/twisted shapes
  • Can distort local clusters
Hard words made easy

Variance = how spread out the data is. High variance = lots of useful difference to learn from. Linear = only straight-line patterns; it can't follow a curve.

t-SNE (nonlinear visualization)

Built for one job: making a 2D picture that reveals clusters. It keeps points that were close together in high dimensions close together in the picture. Great for seeing groups, not for feeding into another model.

Easy example

Like seating guests at a wedding: friends who get along sit together at the same table. t-SNE groups similar data points into visible "tables" (clusters).

Remember

t-SNE is slow on big data and only preserves local structure — the distance between clusters is not meaningful. Use it for exploring/visualizing, not for measuring.

UMAP (the modern upgrade)

Similar goal to t-SNE (a 2D/3D map of the data) but faster and it keeps both local and global structure — so distances between clusters mean more.

t-SNE

  • Slow on big data
  • Local structure only
  • Pretty clusters

UMAP

  • Fast, scales to millions
  • Local + global structure
  • Better cluster spacing
Remember

Both are random (stochastic) — set a seed to get repeatable results. UMAP generally wins on speed and structure.

Autoencoders (neural method)

A neural network that squeezes data through a narrow middle (the bottleneck), then tries to rebuild the original from that squeezed version. The narrow middle is forced to learn the most essential summary.

  • Encoder — compresses the input down.
  • Latent space (bottleneck) — the tiny compressed summary.
  • Decoder — rebuilds the original from the summary.
Easy example

Summarizing a movie in one sentence, then a friend tries to rebuild the plot from your sentence. If they can, your summary kept the important parts. The "one sentence" is the latent space.

Remember

It learns by minimizing reconstruction error (how different the rebuilt output is from the original). A linear autoencoder with no activation = basically PCA. With nonlinear layers, it's more powerful than PCA.

Autoencoder Variants (just know the names)

  • Sparse — forces most of the summary to be zero → cleaner, simpler features.
  • Denoising — give it a messy input, ask for a clean output → learns robust features.
  • Variational (VAE) — learns a distribution, so it can generate new data → a bridge to generative AI.

Choosing the Right Method

  • Need to explain it / interpretability? → PCA
  • Just want a 2D picture of clusters? → t-SNE or UMAP
  • Big dataset? → UMAP or Autoencoder (t-SNE struggles past ~50K points)
  • Feature extraction for another model? → PCA or Autoencoder
Remember — quick cheat

PCA = fast & explainable. t-SNE/UMAP = visualization. Autoencoder = flexible & powerful. PCA is usually the right first try.

The Ethics Bit (likely exam point)

  • Hidden bias — the info you throw away might include protected groups; fairness problems can vanish from view without leaving the model.
  • Structural distortion — 2D plots always distort. Clusters that look separate may overlap in reality.
  • Misleading visuals — a pretty plot can create false confidence. Always ask what was kept and what was lost.

Chapter 7

Neural Networks & Deep Learning Basics

Structured vs Unstructured Data

Deep learning's superpower is handling messy raw data directly.

Structured

  • Neat tables/columns
  • Each feature has clear meaning
  • Classic ML handles it

Unstructured

  • Images, audio, free text
  • Raw pixels mean nothing alone
  • Deep learning shines here
Easy example

A spreadsheet of bird facts = structured. A folder of bird photos = unstructured. Deep learning can learn straight from the photos without you hand-labeling every feature.

Biological vs Artificial Neuron

Artificial neurons are loosely inspired by brain cells, but they're just math — not real simulations.

  • Brain neuron: gets signals, and "fires" if they pass a threshold.
  • Artificial neuron: takes numbers, multiplies each by a weight, adds them up, adds a bias, then passes the result through an activation function.
Easy example

Deciding whether to go outside. Inputs: weather, mood, free time. You weigh each one (weather matters most), add them up, and if the total crosses your "worth it" line, you go. That's a neuron.

Weights & Bias (the learnable parts)

  • Weight — how important an input is. Big weight = big influence.
  • Bias — a nudge that lets the neuron fire even when inputs are zero; shifts the decision line.
Remember

Weights and bias are exactly what training adjusts. "Learning" = finding the best weights and biases.

One Neuron = Logistic Regression

A single neuron with a sigmoid is literally a logistic regression classifier (from Ch 3). Stacking many neurons into layers is what makes a real neural network.

The Multi-Layer Perceptron (MLP)

The basic neural network. Neurons are organized into layers, and every neuron connects to every neuron in the next layer (fully connected). Data flows one way: forward.

  • Input layer — receives the raw features (no math, just passes values in).
  • Hidden layers — the "thinking" layers that learn patterns. More layers = deep network.
  • Output layer — gives the final answer (1 neuron for yes/no, K neurons for K classes).
Hard words made easy

Shallow network = 1–2 hidden layers. Deep network = many hidden layers, each learning a more abstract idea (edges → shapes → faces). That depth is the "deep" in deep learning.

Counting Parameters (common exam trick)

Each connection is one weight; each neuron (except input) has one bias.

  • Weights between two layers = (neurons in layer A) × (neurons in layer B).
  • Biases = one per neuron in the receiving layer.
Easy example

Input 3 → Hidden 4: weights = 3 × 4 = 12, plus 4 biases. Hidden 4 → Output 2: weights = 4 × 2 = 8, plus 2 biases. Total = 12 + 4 + 8 + 2 = 26 parameters.

Why Activation Functions Matter

Without them, stacking layers is pointless — many linear layers collapse into one straight line. Activation functions add non-linearity, letting the network learn complex, curved patterns.

Easy example

Mixing only straight rulers can never draw a circle. You need a bendy tool. Activation functions are that bendy tool — they let the network curve.

Remember

No activation = no matter how many layers, you only get a linear model. Non-linearity is what makes depth worthwhile.

The Three Activation Functions

  • ReLU — keeps positives, turns negatives into 0. Cheap and fast. Default for hidden layers. Risk: "dying ReLU" (neurons stuck at 0).
  • Sigmoid — squeezes to 0–1. Good for a yes/no probability output. Weak in hidden layers (vanishing gradient).
  • Softmax — turns scores into probabilities that add up to 1. Used for the multi-class output.
Remember — the rule of thumb

ReLU in hidden layers · Sigmoid for binary output · Softmax for multi-class output.

Loss Function (how wrong it is)

A score for how bad the predictions are. Training tries to make it as small as possible.

  • MSE (Mean Squared Error) — for regression (number predictions).
  • Cross-Entropy — for classification (category predictions).

Backpropagation (how it learns)

The key learning algorithm. After a guess, it works backward from the error to figure out how much each weight contributed to the mistake, then nudges every weight to do better next time.

  • 1. Forward pass — make a prediction.
  • 2. Loss — measure how wrong it was.
  • 3. Backward pass — trace the blame back to each weight.
  • 4. Weight update — adjust weights a little in the right direction.
Easy example

A basketball shot misses left. You trace back what caused it (wrist angle, power) and adjust each slightly. Repeat thousands of times → you get good. That's backprop.

Remember

Backprop uses the chain rule to assign blame backward. The learning rate controls how big each adjustment is.

Optimizers: SGD vs Adam

The optimizer decides how to update weights using the gradients.

SGD

  • Simple, updates on small batches
  • Often best final result
  • Needs careful tuning

Adam

  • Adapts the step size per weight
  • Fast, works out of the box
  • Great default choice
Remember

Common workflow: start with Adam for quick results, then switch to SGD to squeeze out the best final performance.

Landscape Words (quick definitions)

  • Local minimum — a low point that isn't the lowest overall.
  • Saddle point — flat spot that's not a minimum; momentum helps escape it.
  • Learning rate — the most important setting. Too big → diverges; too small → painfully slow.
  • Overfitting — memorizing training data. Fixes: dropout, L2, data augmentation, early stopping.

The Ethics Bit (likely exam point)

  • Bias enters from skewed training data, biased labels, proxy features (like zip code), and feedback loops.
  • Black box problem — deep networks are hard to explain; you can't easily ask "why?".
  • Solutions — explainability tools (SHAP, LIME), interpretable designs, and clear model documentation. Laws like the EU AI Act demand explanations for high-risk AI.
Remember

Bias and opacity are engineering responsibilities, not afterthoughts.

Chapter 4

Support Vector Machines & Evaluation Metrics

What is an SVM?

A classifier that draws the best dividing line between two groups. Instead of just any line that separates them, it picks the line with the widest gap on both sides.

Easy example

Imagine two groups of kids in a playground. You want to draw a chalk line between them so neither group is crowded against it. SVM finds the line that leaves the most empty space on both sides — the safest divider.

Margin & Support Vectors

  • Margin: the empty gap between the dividing line and the nearest points. SVM makes this gap as wide as possible.
  • Support vectors: the few closest points that "hold up" the line. Only these matter — you could delete every other point and get the same line.
Easy example

Like a road between two neighborhoods. The houses right at the edge of the road decide how wide it can be. Houses far away don't affect it at all — those edge houses are the "support vectors."

Remember

Wider margin = more confident, more reliable on new data. Only the support vectors (closest points) define the boundary.

Hard Margin vs Soft Margin

Hard Margin

  • Allows zero mistakes
  • Needs perfectly separable data
  • Breaks with noise/overlap

Soft Margin

  • Allows a few mistakes
  • Handles messy real data
  • Much more practical
Easy example

Hard margin is a strict teacher who accepts no errors — only works if the class is perfect. Soft margin is a fair teacher who allows a couple of slip-ups so the overall result is better in the real, messy world.

The C Parameter (the strictness dial)

C controls how many mistakes SVM tolerates while drawing the line.

  • Small C → relaxed. Wider gap, allows more mistakes. Can be too simple (underfit).
  • Large C → strict. Narrow gap, allows few mistakes. Can memorize noise (overfit).
Remember

C is a strictness dial. Low C = forgiving & general. High C = strict & risky. The best C is usually found by testing (cross-validation).

The Kernel Trick

Sometimes groups can't be split by a straight line. The kernel trick lets SVM bend the boundary by secretly viewing the data from a higher angle — without doing the heavy math directly.

Easy example

Picture coins scattered on a table — red in the center, blue around the edges. No straight line can separate them. But if you lift the red coins up off the table, you can now slide a flat sheet of paper between red and blue. The kernel does this "lifting" for you.

Common Kernels

  • Linear — a straight line. Best when data is already easy to separate (e.g. text).
  • Polynomial — curved boundaries; captures feature combinations.
  • RBF (Radial Basis Function) — the most flexible, makes smooth blob-like boundaries. The popular default.
Remember

Straight-line data → Linear. Curvy/complex data → RBF is the go-to all-rounder.

SVM Strengths & Weaknesses

Good at

  • High dimensions
  • Clear margins
  • Kernels for curves
  • Memory-light (few support vectors)

Struggles with

  • Very large datasets (slow)
  • Picking C & kernel
  • Hard to interpret
  • No direct probability

The Confusion Matrix

A simple 2×2 table that shows where a classifier was right and wrong. Four boxes:

  • True Positive (TP): said yes, and it was yes ✅
  • True Negative (TN): said no, and it was no ✅
  • False Positive (FP): said yes, but it was no ❌ (false alarm)
  • False Negative (FN): said no, but it was yes ❌ (missed it)
Easy example

A fire alarm. False Positive = it beeps with no fire (annoying). False Negative = real fire but no beep (dangerous). The matrix tracks how often each happens.

Why Accuracy Can Lie

Accuracy = how many you got right overall. It sounds great, but it fails on imbalanced data (when one group is rare).

Easy example

If only 1 in 100 people has a disease, a lazy model that says "healthy" to everyone is 99% accurate — yet it catches zero sick people. High accuracy, totally useless.

Remember

On rare-event problems (disease, fraud, spam), don't trust accuracy alone. Use precision, recall, and F1.

Precision vs Recall

Precision

  • Of the "yes" guesses, how many were right?
  • Punishes false alarms
  • "Don't cry wolf"

Recall

  • Of the real "yes" cases, how many did we catch?
  • Punishes misses
  • "Don't miss any"
Easy example

Precision matters for spam filters — you don't want a real email wrongly trashed. Recall matters for cancer screening — missing a real tumor is far worse than a false alarm you can double-check.

Remember

Precision = "when I say yes, am I right?" Recall = "did I find all the real ones?" Raising one often lowers the other (the trade-off).

F1-Score

One number that balances precision and recall. It stays low if either one is bad, so you can't cheat by maxing only one.

Easy example

Like judging a student on both speed and accuracy. Being super fast but wrong (or perfect but painfully slow) gives a poor combined score. You need both to be decent.

Remember

Use F1 when you care about precision AND recall together — especially on imbalanced data.

ROC Curve & AUC

The ROC curve shows how well a model separates the two classes across all possible cut-off points. AUC (Area Under the Curve) squeezes that into a single score.

  • AUC = 1.0 → perfect separation
  • AUC = 0.5 → useless, just guessing (coin flip)
  • Higher AUC = better. 0.9+ is excellent.
Easy example

AUC is the chance that, if you pick one real "yes" and one real "no" at random, the model ranks the "yes" higher. A perfect model always does (1.0); a clueless one is right half the time (0.5).

Remember

AUC ignores class imbalance, so it's great for rare-event problems. 0.5 = coin flip, 1.0 = perfect.

Cross-Validation (K-Fold)

Instead of testing on just one lucky/unlucky split, you split the data into K parts, train on most and test on the rest, and rotate so every part gets a turn as the test set. Then average the scores.

Easy example

Like getting graded by 5 different teachers instead of 1. One teacher might be unusually easy or harsh; averaging all 5 gives a fairer, more trustworthy grade.

Remember

K-Fold gives a more honest, stable score. Use Stratified K-Fold for imbalanced data so each fold keeps the same class mix (e.g. always includes some rare cases).

Chapter 5

Unsupervised Learning — Clustering

What is Unsupervised Learning?

Here the data has no labels — no answer key. The computer explores on its own to find hidden groups or patterns.

Supervised

  • Has labels (answers)
  • "Learn with a teacher"
  • Predict / classify

Unsupervised

  • No labels
  • "Explore on your own"
  • Find groupings
Easy example

Dump a basket of mixed fruit on a table with no names given. You'd naturally pile them by color or size. That self-made grouping is exactly what clustering does.

Types of Unsupervised Learning

  • Clustering — group similar items (our main focus).
  • Dimensionality reduction — shrink many features into fewer while keeping the meaning.
  • Anomaly detection — spot the odd one out.
  • Generative modeling — learn the data's pattern to create new similar data.
Easy example

Anomaly detection in your bank: it learns your normal spending, then flags a sudden $5,000 charge abroad as suspicious.

Distance = Similarity

Clustering measures how "alike" two points are by the distance between them. Close together = similar = likely same group.

  • Euclidean — straight-line distance (the usual one).
  • Manhattan — distance along a grid, like city blocks.
  • Cosine — compares direction/angle, great for text.
Easy example

Euclidean = a bird flying straight to a point. Manhattan = a taxi that can only drive along streets, turning at corners.

Always Scale Your Features

Before clustering, put all features on the same scale. Otherwise a big-numbered feature bullies the small ones and ruins the grouping.

Easy example

Comparing people by age (0–100) and salary (0–100,000). Without scaling, salary's huge numbers dominate and age is ignored. Scaling makes them count equally.

Remember

Distance-based methods are very sensitive to scale. Always scale first (StandardScaler) — this is a classic exam point.

The 3 Clustering Methods

Three different "philosophies" for grouping data:

  • K-Means — splits data into K round blobs around centers.
  • Hierarchical — builds a tree of groups; no need to pick K upfront.
  • DBSCAN — groups by density; handles odd shapes and noise.

K-Means Clustering

You tell it how many groups (K) you want. It places K centers, assigns each point to its nearest center, then moves the centers to the middle of their points — and repeats until settled.

Easy example

Setting up 3 food stalls in a park. You want each stall in the middle of its crowd. You guess 3 spots, see who goes where, move each stall to the center of its group, and repeat until nobody switches stalls.

Pros

  • Simple & fast
  • Scales to big data

Cons

  • Must pick K
  • Assumes round blobs
  • Sensitive to outliers
Remember

K-Means needs you to choose K in advance and assumes round, similar-sized clusters.

Choosing K: Elbow & Silhouette

  • Elbow method: plot the error as K grows. The "elbow" bend — where adding more groups stops helping much — is a good K.
  • Silhouette score: rates how tight and well-separated the clusters are. Ranges −1 to 1; higher is better.
Easy example

Elbow = like adding study hours: going from 1→3 hours helps a lot, but 8→9 hours barely moves your grade. That flattening point is the sweet spot.

Remember

Elbow = find the bend. Silhouette = higher is better. They often agree on the same K.

Hierarchical Clustering

Builds a tree of groups (a "dendrogram"). The common style (agglomerative) starts with every point alone, then keeps merging the two closest groups until all are joined.

Easy example

A family tree in reverse: cousins merge into families, families into clans, clans into one big group. You can "cut" the tree at any height to decide how many groups you want — after building it.

Remember

Big advantage: no need to pick K first — just cut the tree where you like. Downside: slow on large datasets.

Reading a Dendrogram & Linkage

In the tree: each merge's height = how far apart those groups were. To pick clusters, draw a horizontal cut — look for the tallest gap with no merges, which suggests a natural number of groups.

Linkage = the rule for measuring distance between groups:

  • Single — closest pair (can cause stringy "chains").
  • Complete — farthest pair (tight, compact groups).
  • Average — average of all pairs (a balanced middle).
  • Ward's — keeps clusters neat by minimizing messiness. Usually the best default.

DBSCAN (density-based)

Groups points that are packed closely together and labels lonely points as noise. It doesn't need you to pick K and can find weird, non-round shapes.

Three kinds of points:

  • Core point — has lots of close neighbors (in a crowd).
  • Border point — near a crowd but not crowded itself (edge).
  • Noise point — alone, far from everyone (outlier).
Easy example

People at a festival. Dense clusters = the crowds at each stage. A person wandering alone in an empty field = noise. DBSCAN finds the crowds no matter their shape and ignores the loners.

Remember

DBSCAN shines for odd shapes + noise and needs no K. But it struggles when clusters have very different densities and in high dimensions.

K-Means vs DBSCAN (common exam question)

Pick K-Means when

  • Clusters are round & similar size
  • You know roughly how many
  • Data is large (speed)

Pick DBSCAN when

  • Clusters are odd shapes
  • There's noise/outliers
  • You don't know K

Evaluating Clusters (no labels!)

Without an answer key, we judge clusters two ways:

Intrinsic

  • Uses data only, no labels
  • Silhouette, Davies-Bouldin
  • Works in real life

Extrinsic

  • Compares to true labels
  • ARI, NMI
  • Only for testing/benchmarks
  • Silhouette — higher is better (tight + separated).
  • Davies-Bouldin — lower is better (0 = perfect).
Remember

In real unsupervised work you usually have no labels → rely on intrinsic metrics. Silhouette high = good; Davies-Bouldin low = good. Use several metrics + common sense, not just one.

Ethics: Audit Your Clusters

Clustering can accidentally pick up bias hidden in the data. If past data was unfair, the groups can quietly repeat that unfairness.

Easy example

A hiring tool that clusters applicants might group them in ways that line up with gender or background — leading to unfair shortlists. Always check the groups before trusting them.

Remember

Hidden structure can encode bias. Audit cluster fairness before deploying — a likely ethics exam point (CLO 3.1).

Chapter 1: Introduction to Operating Systems

Start here if you need the core ideas: what the operating system does, how it manages hardware, and how the computer stays protected while many programs run.

Computer System Components

  • Hardware: CPU, RAM, storage, and input/output devices.
  • Operating system: coordinates the use of hardware.
  • Application programs: browsers, editors, compilers, games.
  • Users: people or other systems using the machine.

Device Controller vs Driver

  • Device controller: hardware that manages a specific device.
  • Driver: OS software that talks to the controller.
  • The driver lets the OS use very different devices through a more standard interface.

1. What an Operating System Is

  • An operating system is system software that manages hardware and provides services to programs and users.
  • It works as the layer between applications and the physical machine.
  • It is a resource allocator because it decides how CPU time, memory, storage, and I/O devices are shared.
  • It is a control program because it supervises execution and prevents improper or unsafe use of hardware.
Exam shortcut: the OS mainly does resource management and control of execution.

2. Interrupts and Traps

  • An interrupt makes the CPU pause the current work and transfer control to the operating system.
  • Typical steps are: save current state, identify the source, run the interrupt service routine, restore the state, then continue.
  • Hardware interrupts come from devices such as disks, keyboards, or network cards.
  • Traps / exceptions come from the CPU or running program, such as divide-by-zero, invalid memory access, or a system call.

DMA

  • Direct Memory Access moves a block of data between a device and memory with little CPU involvement.
  • The CPU only sets up the transfer, then the DMA controller handles the copy.
  • Main benefit: fewer interrupts and better performance for large transfers.

Timer Interrupt

  • The timer guarantees the OS gets control of the CPU back.
  • It prevents one program from running forever and blocking the system.
  • Only the OS can configure the timer because that is a privileged operation.

3. Boot Process and Storage Hierarchy

  • The bootstrap program starts first from ROM or firmware.
  • It initializes the machine and loads the kernel into RAM.
  • After the kernel starts, background services and user-facing system services begin running.
  • Storage hierarchy idea: registers are fastest, then cache, then RAM, while SSD/HDD are slower but keep data after power off.
  • Important exam rule: the CPU can directly work with registers and main memory, not with disk addresses.

4. Multiprogramming and Multitasking

  • Multiprogramming keeps several jobs in memory so the CPU stays busy when one job waits for I/O.
  • Multitasking / timesharing switches quickly between jobs to give users fast response.
  • Multiprogramming focuses on CPU utilization, while multitasking focuses more on responsiveness too.
  • The fast switching gives the illusion that many jobs run at the same time.

5. User Mode vs Kernel Mode

  • User mode is limited and is where normal applications run.
  • Kernel mode is privileged and gives full access to hardware and protected instructions.
  • The mode bit tells the processor which mode it is in.
  • User mode protects the system by stopping applications from directly changing hardware state or other programs' memory.
  • Switching from user mode to kernel mode usually happens through interrupts, traps, or system calls.
Big picture: chapter 1 explains how the OS controls the hardware safely and why normal programs cannot do everything directly.

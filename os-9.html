Chapter 9: Main Memory

This chapter moves from basic address protection to contiguous allocation, fragmentation, paging, TLBs, and large page-table structures.

1. Basic Memory Facts

  • A program must be brought from disk into memory before it can execute.
  • The CPU can directly access registers and main memory, but not disk locations directly.
  • Cache exists between CPU registers and main memory to reduce delay.
  • Memory protection is necessary so one process cannot access another process's memory or kernel memory.

2. Base Register and Limit Register

  • The base register stores the smallest legal physical address for the process.
  • The limit register stores the size of the legal address range.
  • Every user-mode memory access must be checked against these values.
  • Loading these registers is privileged, so only the OS can do it.

3. Address Binding and MMU

  • Address binding can happen at compile time, load time, or execution time.
  • Logical address is generated by the CPU.
  • Physical address is the actual location in RAM.
  • The MMU maps logical addresses to physical addresses during execution-time binding.
  • This separation gives protection, relocation flexibility, and the foundation for virtual memory later.

4. Contiguous Allocation

  • Each process gets one continuous block of physical memory.
  • Free blocks are called holes.
  • Main allocation strategies are first-fit, best-fit, and worst-fit.
  • First-fit chooses the first hole big enough, best-fit chooses the smallest hole big enough, and worst-fit chooses the largest hole.

5. Fragmentation

  • External fragmentation means enough total free memory exists, but it is split into separate holes so one large request still cannot fit.
  • Internal fragmentation means the allocated block is larger than the process actually needs, so some space inside the block is wasted.
  • Compaction can reduce external fragmentation by moving processes together and combining free space.
  • Compaction is expensive and needs dynamic relocation support.

6. Paging

  • Physical memory is divided into fixed-size frames.
  • Logical memory is divided into fixed-size pages.
  • The page table maps page numbers to frame numbers.
  • Paging removes external fragmentation because pages can be placed in any free frame.
  • Paging may still leave some internal fragmentation.

7. TLB and Protection Bits

  • A TLB is a fast cache holding some recent page-table entries.
  • TLB hit means the translation is found quickly.
  • TLB miss means the system must read the page table from memory.
  • Valid-invalid bits show whether a page belongs to the process address space.
  • Protection bits help enforce read-only or read-write permissions.

8. Large Page-Table Structures

  • Large logical address spaces make a single flat page table too large.
  • Main solutions are hierarchical page tables, hashed page tables, and inverted page tables.
  • Hierarchical paging splits the page table into smaller levels.
  • Hashed tables are useful for large address spaces, while inverted page tables reduce memory used for page-table storage.
Quick summary: contiguous allocation is simple but suffers from fragmentation, while paging gives flexible placement and supports modern memory management.

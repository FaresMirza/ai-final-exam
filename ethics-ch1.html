Chapter 1

Software Engineering Professional Practice and Code of Ethics

Why a Code of Ethics Exists

Software engineers build systems that directly affect people, organizations, and society, so their decisions must be guided by more than technical convenience.

  • Software work can improve lives, but it can also create privacy loss, unsafe systems, unfair decisions, or public harm.
  • The code exists to make software engineering a beneficial and respected profession.
  • Ethics helps when the law is silent, unclear, or too weak to guide the right decision.

What Professional Practice Means

  • Professional practice means acting with honesty, competence, care, and accountability.
  • A software engineer is not judged only by whether the code runs, but also by whether the work is responsible.
  • The ACM/IEEE code is a joint professional standard used to define expected ethical behaviour.
Exam focus

Professional behaviour in software engineering means doing technically sound work in a way that is honest, safe, and fair.

Key Responsibility Areas

  • Confidentiality means protecting employer and client information appropriately.
  • Competence means not accepting work you are not qualified to perform responsibly.
  • Intellectual property means respecting copyright, patents, and ownership of software assets.
  • Computer misuse means not using technical skills to abuse systems, data, or access rights.

The First Four Principles: What They Mean

Public and Client

  • Public: protect users, society, privacy, safety, and the environment
  • Client and Employer: serve them honestly, but never against the public interest

Product and Judgment

  • Product: produce software of high quality, properly tested, and professionally built
  • Judgment: keep professional independence and do not hide risks or defects

The Last Four Principles: What They Mean

  • Management: leaders should promote ethical development and maintenance practices.
  • Profession: engineers should protect the integrity and reputation of software engineering.
  • Colleagues: engineers should be fair, supportive, and respectful to others they work with.
  • Self: engineers should keep learning and improving their ethical and technical practice.
Exam shortcut

If a final-exam question asks what the code is really trying to protect, the answer is usually: the public good, professional honesty, software quality, and responsible behaviour.

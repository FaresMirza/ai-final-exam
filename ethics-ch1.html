Chapter 1

Software Engineering Professional Practice and Code of Ethics

Why a Code of Ethics Exists

Software engineers affect commerce, health, education, government, and daily life, so their work can create major good or major harm.

  • Software engineers influence how systems are analysed, designed, built, tested, and maintained.
  • The code exists to help make software engineering a beneficial and respected profession.
  • Ethics gives guidance when legal compliance alone is not enough.

How the Code Was Developed

  • The IEEE Board of Governors started a steering effort in 1993.
  • ACM then joined through a joint steering committee in 1994.
  • The committee aimed to define a body of knowledge, recommended practices, ethical standards, and curriculum guidance.
Remember

The code was a joint professional effort, not just one organization's opinion.

Ethics Goes Beyond Technical Skill

  • Professional software work includes moral responsibility, not only technical correctness.
  • Ethical behaviour means more than obeying the law; it means following morally sound principles.
  • Respect as a professional depends on honesty, responsibility, and good judgment.

Core Responsibility Areas

Professional limits

  • Respect confidentiality
  • Do not overstate competence
  • Protect employer and client IP

Misuse warnings

  • Do not misuse computers
  • Do not spread malware or abuse access
  • Avoid unethical or illegal software use

The Eight ACM/IEEE Principles

  • Public, Client and Employer, Product, Judgment
  • Management, Profession, Colleagues, Self
  • The strongest theme is that software should serve the public interest.
  • Engineers should approve software only when they have good reason to believe it is safe and properly tested.
Exam shortcut

If you forget the details, remember that the code keeps pulling decisions back toward safety, honesty, fairness, and the public good.

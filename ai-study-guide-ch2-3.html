Chapter 2

Supervised Learning — Regression

Supervised Learning Basics

The computer learns from labeled examples (data where we already know the answer), so it can predict answers for new data.

  • Features (X): the inputs / clues (e.g. size of a house).
  • Target (y): the answer we want (e.g. the price).
Easy example

Like studying past exams with the answer key. After enough practice, you can answer new questions you've never seen.

Regression vs Classification

Regression

  • Predicts a number
  • Price, temperature, salary
  • "How much?"

Classification

  • Predicts a category
  • Spam / not spam
  • "Which group?"
Remember

Regression = continuous numbers. Classification = labels/categories. This chapter is all about regression.

The Workflow (5 steps)

  • 1. Collect labeled data
  • 2. Split it (usually 70–80% to train, 20–30% to test)
  • 3. Choose a model
  • 4. Train the model on the training data
  • 5. Evaluate on the test data it has never seen
Remember

Always test on unseen data. Testing on training data is like grading a student with the same questions they practiced — not honest.

Linear Regression

Fits a straight line through the data to capture the trend. One feature = simple linear regression; many features = multiple linear regression (a flat "plane" instead of a line).

Easy example

Salary vs experience: more years → higher salary. The line lets you guess the salary for any number of years.

Cost Function (how "wrong" the model is)

The model measures error by looking at the gap between its prediction and the real value. We square those gaps so they're always positive and big mistakes get punished more. Training = making this total error as small as possible.

Easy example

Think of throwing darts: the cost is how far you missed the bullseye. The model keeps adjusting to miss by less.

The 5 Assumptions (linear regression only works well if these hold)

  • 1. Linearity — the real relationship is roughly a straight line.
  • 2. Independence — one data point doesn't depend on another.
  • 3. Homoscedasticity — the error size stays about the same everywhere (don't get bigger for large values).
  • 4. Normality of errors — errors follow a normal "bell curve".
  • 5. No multicollinearity — features shouldn't be copies of each other.
Hard words made easy

Homoscedasticity = "equal scatter." Errors are evenly spread, not a fan/cone shape.
Multicollinearity = two features saying the same thing, e.g. size in feet AND size in meters. The model gets confused about which one matters.

Remember

If assumptions break → unreliable predictions and misleading results.

Polynomial Regression

When data is curved, a straight line fails. Polynomial regression adds powers (x², x³…) so the line can bend and follow curves.

Easy example

Car fuel efficiency vs speed: bad at very low speed, best in the middle, bad at high speed — a U-shape. A curve fits this; a straight line can't.

Remember

Underfit = too simple, misses the pattern. Overfit = too complex, memorizes noise. The sweet spot in the middle is best. This is the bias-variance tradeoff.

Gradient Descent (how the model learns)

A step-by-step method to find the lowest error. It keeps taking small steps "downhill" until it reaches the bottom (minimum error).

Easy example

Like walking down a foggy mountain — you can't see the bottom, so you just keep stepping in the steepest downhill direction until you can't go lower.

Three types:

  • Batch — uses ALL data each step. Stable but slow on big data.
  • Stochastic (SGD) — uses ONE example each step. Fast but noisy/jumpy.
  • Mini-batch — uses a small group (e.g. 32–256). Best of both → the industry standard.
Remember — Learning Rate

The step size. Too big → overshoots and never settles. Too small → super slow. Just right → reaches the bottom quickly.

Measuring Performance (metrics)

  • MSE / RMSE — average error; punishes big mistakes hard. RMSE is in the same units (e.g. dollars), so it's easy to read.
  • MAE — average error treating all mistakes equally; more robust to outliers.
  • R² score — how much of the pattern the model explains. 1 = perfect, 0 = useless. (R² = 0.85 means it explains 85%.)
Remember

Big errors matter most → use RMSE. Outliers shouldn't dominate → use MAE. Comparing across datasets → use R².

Chapter 3

Logistic Regression, Decision Trees & Random Forest

From Regression to Classification

Now we predict categories, not numbers: spam or not, sick or healthy, approved or rejected.

Binary

  • Only 2 outcomes
  • Spam / Ham
  • Fraud / Legit

Multi-class

  • 3+ outcomes
  • Digits 0–9
  • Pos / Neutral / Neg

Why Linear Regression Fails Here

A straight line can give answers like 2.3 or −0.5 — but a probability must stay between 0 and 1. We need something that squeezes any number into that range.

The Sigmoid Function

An S-shaped curve that turns any number into a probability (0 to 1). That's the heart of logistic regression.

  • Output always between 0 and 1
  • At input 0 → output is 0.5 (the middle)
  • Big positive input → near 1; big negative → near 0
Easy example

Think of a dimmer switch: no matter how hard you push, the light stays between fully off (0) and fully on (1) — never beyond.

Logistic Regression Model

It outputs the probability that something belongs to class 1. If that probability is above the threshold (usually 0.5), we predict class 1.

Easy example

Spam filter says "92% spam" → above 0.5 → mark as spam. A "51%" answer means it's unsure.

Odds & Log-Odds (don't panic)

Just another way to express chance.

  • Odds = how much more likely "yes" is than "no". Odds of 3 → 3× more likely to happen.
  • Log-odds = the log of the odds. Positive → class 1 more likely; negative → class 0 more likely.
Easy example

"This team has 2-to-1 odds of winning" = twice as likely to win as lose. Same idea, just gambling language.

Decision Boundary

The dividing line between classes. On one side it predicts class 1, on the other class 0. For logistic regression this line is straight — which limits it for complex shapes.

Remember

The threshold is usually 0.5 but you can change it depending on which mistake is worse.

Binary Cross-Entropy (the loss)

The error measure for classification. It punishes confident wrong answers heavily. Saying "99% spam" about a real email = huge penalty.

Easy example

Like a confident exam answer: if you're 100% sure and wrong, it hurts more than a hesitant guess.

Regularization (stops overfitting)

Adds a penalty for overly large weights so the model stays simple and general.

L2 (Ridge)

  • Shrinks weights toward 0
  • Keeps all features
  • Smooth reduction

L1 (Lasso)

  • Pushes some weights to exactly 0
  • Deletes useless features
  • = feature selection

Reading the Weights (coefficients)

  • Positive weight → pushes toward class 1 (e.g. higher income → loan approved).
  • Negative weight → pushes toward class 0 (e.g. higher debt → rejected).
  • Bigger size → stronger influence on the decision.

Logistic Regression — Pros & Cons

Strengths

  • Gives probabilities
  • Fast & efficient
  • Easy to interpret

Weaknesses

  • Only straight boundaries
  • Weak on complex patterns
  • Sensitive to feature scaling

Decision Trees

A flowchart of yes/no questions. You follow the answers down until you reach a final prediction at the bottom (a "leaf").

  • Root = the starting question (all data)
  • Internal nodes = more questions
  • Leaves = final answers
Easy example

Like the game "20 Questions": "Is it bigger than a cat? Does it fly?" Each answer narrows it down to the final guess.

Remember

Trees are very easy to read (each path = a clear rule) and handle curved/complex data without a straight line.

How a Tree Picks Questions: Entropy & Information Gain

  • Entropy = how mixed/messy a group is. All one class = pure = 0. Half-and-half = most messy.
  • Information Gain = how much a question cleans up the mess. The tree picks the question with the highest gain.
  • Gini impurity = another way to measure messiness; faster to compute so it's often preferred. Both give similar trees.
Easy example

Sorting a mixed bag of red and blue balls. A good question splits them into a "mostly red" pile and a "mostly blue" pile. That big cleanup = high information gain.

Tree Decision Boundaries

Trees split space into boxes (rectangles). Boundaries are always straight and at right angles — a step/staircase shape, not a smooth curve.

Controlling Overfitting in Trees

Trees love to memorize. Two fixes (note: regularization doesn't apply to trees):

  • Pre-pruning / limit growth — set rules early, e.g. max depth or minimum samples to split.
  • Pruning — let it grow fully, then cut branches that don't help on validation data.

Limits of Single Trees

  • Overfitting — deep trees memorize
  • High variance — small data change → very different tree
  • Greedy — picks best split now, may miss the best overall

Random Forest (the fix)

Instead of one tree, build many trees and let them vote. The majority answer wins. This cancels out individual mistakes.

  • Bagging — each tree trains on a random sample of the data
  • Feature randomness — each split looks at only some features
  • Voting — combine all trees for the final answer
Easy example

Asking ONE friend for advice can be biased. Asking 100 different friends and going with the majority is far more reliable. That's a forest.

Remember

Random forest = lower variance + more robust + less overfitting than a single tree. Performance plateaus around 100–500 trees.

One-vs-Rest (multi-class trick)

Turns a multi-class problem into several yes/no problems — one classifier per class ("Is it class A or not?"). The class with the highest probability wins.

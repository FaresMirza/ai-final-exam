Chapter 5: CPU Scheduling

This chapter is one of the most calculation-heavy topics. Focus on what each scheduler chooses, whether it is preemptive, and how to compute waiting, response, and turnaround times.

Core Formulas

  • TAT = Completion Time - Arrival Time
  • WT = Turnaround Time - Burst Time
  • Response time = First time process gets CPU - Arrival Time

Scheduler vs Dispatcher

  • Scheduler decides which process runs next.
  • Dispatcher performs the actual CPU handoff.
  • Dispatch latency is the overhead needed to stop one process and start another.

1. Scheduling Criteria

  • Maximize CPU utilization and throughput.
  • Minimize turnaround time, waiting time, and response time.
  • Turnaround time means from arrival to completion.
  • Waiting time means time spent in the ready queue.
  • Response time means how quickly the process gets its first response.

Preemptive Scheduling

  • The OS can take the CPU away from a running process.
  • It usually gives better responsiveness.
  • It introduces more overhead because switching happens more often.

Non-Preemptive Scheduling

  • The process keeps the CPU until it finishes or blocks.
  • It is simpler but may make short jobs wait behind long ones.
  • It is common in textbook comparisons because the execution order is easier to follow.

2. FCFS, SJF, and SRTF

AlgorithmMain RuleImportant Point
FCFSArrival orderSimple but suffers from convoy effect
SJFShortest burst time firstOptimal average waiting time if burst lengths are known
SRTFShortest remaining time firstPreemptive version of SJF
Useful memory trick: SJF looks at total burst, while SRTF keeps checking the remaining burst.

3. Round Robin and Priority Scheduling

  • Round Robin gives each process a time quantum, then moves unfinished work to the back of the queue.
  • If the quantum is too large, RR becomes similar to FCFS.
  • If the quantum is too small, context-switch overhead becomes too high.
  • Priority scheduling runs the highest-priority process first.
  • Main problem with priority scheduling is starvation; the standard fix is aging.

4. Multilevel Queue and Multilevel Feedback Queue

  • Multilevel queue places processes permanently into different queues by type.
  • Multilevel feedback queue allows processes to move between queues depending on behavior.
  • Feedback queues are more flexible and can react to whether a process behaves like I/O-bound or CPU-bound work.
  • Feedback queues can also help reduce starvation better than fixed multilevel queues.

5. Solving Scheduling Questions

  • Always draw or imagine the Gantt chart first.
  • Check arrival times before deciding which process is eligible to run.
  • For RR, it is often easiest to compute completion time first, then TAT, then WT.
  • If two processes tie, many exercises use FCFS as the tie-break rule.
  • If the algorithm is preemptive, remember that the process may appear in several separated time blocks.
Fast memory summary: FCFS = arrival order, SJF = shortest burst, SRTF = shortest remaining, RR = time quantum, Priority = highest priority first.

CPU-Burst and I/O-Burst Cycle

  • Processes usually alternate between computing and waiting for I/O.
  • I/O-bound processes have many short CPU bursts.
  • CPU-bound processes have fewer but longer CPU bursts.
  • This difference helps explain why some algorithms favor interactive jobs more than long computation jobs.

When Scheduling Decisions Happen

  • When a running process blocks for I/O.
  • When a running process is preempted and moved back to ready.
  • When a waiting process becomes ready again.
  • When a process terminates.

6. Extra Notes on the Main Algorithms

  • FCFS is easy but suffers from the convoy effect, where short jobs wait behind one long job.
  • SJF is optimal for average waiting time, but the OS usually has to estimate the next CPU burst instead of knowing it exactly.
  • SRTF can improve average waiting time further because a newly arrived short job can preempt the current one.
  • Round Robin is designed for time sharing, so it is usually better for response time than for minimum average turnaround time.

7. Priority, Aging, and Queue Design

  • Priority scheduling can be either preemptive or non-preemptive depending on the rule used.
  • Starvation happens when a low-priority process keeps waiting because higher-priority work keeps arriving.
  • Aging solves this by gradually increasing the priority of long-waiting processes.
  • In a fixed multilevel queue, processes stay in one queue permanently, but in a multilevel feedback queue they can move between queues based on behavior.

9. Solved-Example Mindset

  • Write all arrival times and burst times clearly before choosing any process.
  • For preemptive algorithms, re-check the ready queue whenever a new process arrives.
  • For RR, follow the queue rotation carefully and stop each slice at the quantum or process completion.
  • After drawing the Gantt chart, compute completion time first, then turnaround time, then waiting time.
  • This is the exact pattern used in most scheduling questions from slides and exams.
One safe order for calculations: Gantt chart → Completion Time → TAT → WT → average values.
